# Waline 评论评分系统重构规范

## 概述

重构 Waline 评论系统的评分功能，删除现有不工作的实现，重新设计并实现一个可靠的评分机制。

## 问题分析

当前评分系统存在的问题：
1. **提交拦截失败**：点击提交按钮时，`rating:*` 前缀未能正确添加到评论内容中
2. **DOM 结构不匹配**：Waline v3 可能使用 contentEditable div 而非 textarea
3. **事件监听时机错误**：可能在 DOM 元素尚未完全渲染时就尝试绑定事件

## 新设计方案

### 1. 评分输入组件
- 位置：评论输入面板顶部，昵称/邮箱输入框之前
- 样式：5 颗星星，类似淘宝评分
- 交互：点击选择评分，悬停预览效果
- 提示文字："你认为这家店怎么样"

### 2. 评分数据存储
- 使用 Waline 内置的 `meta` 字段存储评分（如果支持）
- 或者在评论内容开头添加 `rating:*` 标记

### 3. 评分显示
- 在评论内容前显示星星评分组件
- 样式符合 MD3 设计规范

## 技术实现

### 方案 A：使用 Waline 自定义字段（推荐）
Waline 支持通过 `meta` 传递自定义数据，可以在初始化时配置自定义字段。

### 方案 B：内容前缀方式
在评论提交前，将评分值作为前缀添加到评论内容中。

### 最终方案
采用 **方案 B**，但使用更可靠的方式：
1. 使用 MutationObserver 监听 Waline 编辑器的 DOM 变化
2. 找到实际的输入元素（textarea 或 contentEditable div）
3. 在用户输入时实时同步评分前缀

## 文件修改清单

| 文件 | 修改内容 |
|------|---------|
| `www/app.js` | 删除旧评分代码，重新实现评分逻辑 |
| `www/styles.css` | 保留评分样式，可能需要微调 |

## 验收标准

1. 用户可以选择 1-5 星评分
2. 提交评论时评分值被正确存储
3. 评论列表中显示星星评分
4. 评分组件样式符合 MD3 设计规范
